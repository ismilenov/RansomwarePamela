ModMaker - Administration of a module

Under the Administration tab of the configuration of the module is determined. The following settings can be assigned.

Login for editors
A user group of the user module can be defined as editors, giving it access to the administrator of the module.

Set write permissions
The display of visitor input mask can be limited to user groups in the user module.

Multilingual input
The module is to be used in several languages. For this, a language selection list is created. If the sides are supposed to be the same for all languages, a field for each language is set alternatively per text input.

Offer categories
The topics and/or contributions of the module are accessible via categories. Categories have separate templates and can be provided with image and/or description. Nesting as a category tree as well as the mapping from the set menu are possible.

Categories as entry points for multi-page support
Categories can be used as entry points to use a module on different pages. When this item is selected, the variable "basecat" must be set in the page - more infos.

Offer topics
When this item is selected contributions are grouped under a theme. For the different representation of the thread start contribution the templates are topicslist_startdata.tpl and topic_startdata.tpl available.

Create option groups
Option groups are free customizable lists that can be printed in records. The principle is the same as checkboxes, only more flexible. If one has for example a catalog of products made of wood, metal and plastic could be an option group different wood grains, an option group various metal surfaces and other colors of plastics map - including images. Depending on the product the appropriate option group is then selected in the Administration and mapped in the record.
