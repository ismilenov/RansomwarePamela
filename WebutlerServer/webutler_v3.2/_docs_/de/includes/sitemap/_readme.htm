Modul Sitemap

Die Sitemap für die Homepage wird automatisch erzeugt und nach einem angegebenen Zeitpunkt automatisch aktualisiert (Variable $revisit_after in der Datei "/settings/sitemaps.php").

Welche URLs werden in die Sitemap aufgenommen?

Alle Seiten aus dem Verzeichnis "/content/pages" sowie alle Links angegebener Module. In der Datei "/settings/sitemaps.php" kann eingestellt werden welche Module aufgenommen werden sollen - weitere Infos.

Sitemap-Links für ein eigenes Modul

Ihr Modul muß sich im Verzeichnis "/modules" befinden. Im Modulverzeichnis muß die Datei sitemap.php und in der Datei die Funktion "modulessitemap_ __MODUL_VERZEICHNISNAME__ " existieren. Die Funktion muß ein Array mit den URLs des Modules liefern.

Beispiel:
Array(
[0] => 'http://meineseite.de/module-site1.htm',
[1] => 'http://meineseite.de/module-site2.htm',
[2] => 'http://meineseite.de/module-site3.htm'
)

Meta-Tags

Seitentitel, Description und Keywords können bequem über den Button Dokument-Eigenschaften des Editors eingegeben werden.
Wenn diese Felder in einer ModMaker Modul-Administration aktiviert wurden, dürfen Description und Keywords nicht in der HTML-Seite existieren. Die Tags erscheinen sonst doppelt.

Robots

In HTML-Seiten kann der Meta-Tag Robots nur über den Quelltext gesetzt werden. Am besten setzt man ihn direkt in der Layoutdatei. ModMaker Module setzen den Meta-Tag Robots automatisch, dann darf er nicht in der HTML-Seite existieren. Kategorien-, Themen- und Beitragsseiten bekommen 'index,follow', Kopien (wenn angelegt) 'noindex,follow'.

Canonical-Tag

In alle Seiten und ModMaker-Modulseiten wird automatisch der Canonical-Tag zur Vermeidung von Double-Content eingefügt.
